# 1. `SYS_TABLES`表

|      列名      |                          描述                           |
|:------------:|:-----------------------------------------------------:|
|    `NAME`    |                         表的名称                          |
|     `ID`     | 在InnoDB存储引擎中,每个表都有一个唯一的ID (MySQL8.0中,该列改名为`TABLE_ID`) |
|   `N_COLS`   |                       该表拥有列的个数                        |
|    `TYPE`    |               表的类型,记录了一些文件格式/行格式/压缩等信息                |
|   `MIX_ID`   |                        已过时,忽略                         |
|  `MIX_LEN`   |                       表的一些额外的属性                       |
| `CLUSTER_ID` |                        未使用,忽略                         |
|   `SPACE`    |                      该表所属表空间的ID                       |

MySQL8.X中,该表对用户的展示为:数据库`information_schema`的`INNODB_TABLES`表

`SYS_TABLES`表有2个索引:

- 以`NAME`列为主键的聚簇索引
- 以`ID`列建立的二级索引

注: `information_schema.INNODB_TABLES`表没有这2个索引
