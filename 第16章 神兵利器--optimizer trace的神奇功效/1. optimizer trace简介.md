# 1. `optimizer trace`简介

在MySQL5.6及之后的版本中,MySQL的设计者提出了`optimizer trace`功能,该功能可以方便的查看优化器生成执行计划的整个过程.

这个功能的开启与关闭由系统变量`optimizer_trace`决定:

```
mysql> SHOW VARIABLES LIKE 'optimizer_trace';
+-----------------+--------------------------+
| Variable_name   | Value                    |
+-----------------+--------------------------+
| optimizer_trace | enabled=off,one_line=off |
+-----------------+--------------------------+
1 row in set (0.04 sec)
```

- `enabled`: 表示是否开启`optimizer trace`功能,默认值为`off`,即关闭.
- `one_line`: 表示是否将输出结果格式化为1行,默认值为`off`,即不格式化为1行.保持该选项值为`off`即可.

注: `optimizer_trace`是会话级(SESSION level)动态系统变量,不是全局持久化变量.无法通过直接通过MySQL配置文件来设置,只能通过`SET`命令来设置.

```
mysql> SET optimizer_trace="enabled=on";
Query OK, 0 rows affected (0.00 sec)
```

```
mysql> SHOW VARIABLES LIKE 'optimizer_trace';
+-----------------+-------------------------+
| Variable_name   | Value                   |
+-----------------+-------------------------+
| optimizer_trace | enabled=on,one_line=off |
+-----------------+-------------------------+
1 row in set (0.00 sec)
```

然后输入想要查看优化过程的查询语句,**当该查询语句执行完成后,就可以到`information_schema`数据库
下的`OPTIMIZER_TRACE`表中查看完整的优化过程**(也可以不真正执行查询语句,仅使用`EXPLAIN`来查看语句的执行计划).
`OPTIMIZER_TRACE`表有4列,分别为:

- `QUERY`: 表示输入的查询语句
- `TRACE`: 表示优化过程的JSON格式文本
- `MISSING_BYTES_BEYOND_MAX_MEM_SIZE`: 

    在执行计划的生成过程汇总可能会输出很多内容,若超过某个限制,则多余的文本将不会被显示,这个字段表示被忽略的文本字节数

- `INSUFFICIENT_PRIVILEGES`: 

    表示是否有权限查看优化过程,默认值是0,表示有权限查看执行计划的生成过程.只有某些特殊情况下才会是1,暂时不关心该字段的值

使用`optimizer trace`功能的完整步骤如下:

- 步骤1. 打开`optimizer trace`功能(默认情况下是关闭的)

    ```
    SET optimizer_trace="enabled=on";
    ```

- 步骤2. 输入要查看优化过程的查询语句并执行
- 步骤3. 从`information_schema.OPTIMIZER_TRACE`表中查看上一个查询的优化过程

    ```
    SELECT * FROM information_schema.OPTIMIZER_TRACE;
    ```
    
    或
    
    ```
    SELECT * FROM information_schema.OPTIMIZER_TRACE\G;
    ```
  
    使用`\G`命令可以将结果集以垂直格式输出,更容易阅读

- 步骤4. 若还要观察其他语句执行的优化过程,重复步骤2和步骤3即可
- 步骤5. 当停止查看语句的优化过程时,把`optimizer trace`功能关闭

    ```
    SET optimizer_trace="enabled=off";
    ```
  
例如:

```sql
SELECT *
FROM single_table
WHERE 
    key1 > 'z'
AND
    key2 < 1000000
AND
    key3 IN ('a', 'b', 'c')
AND
    common_field = 'abc';
```
