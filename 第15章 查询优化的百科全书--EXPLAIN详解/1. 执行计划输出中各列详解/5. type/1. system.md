# 1. `system`

若:

- 表中只有1条记录
- 且该表使用的存储引擎的统计数据是精确的
  - 比如MyISAM/Memory

则对该表的访问方法就是`system`.

例如: 新建一个MyISAM表,并为其插入一条记录:

```
mysql> CREATE TABLE t_myisam (a INT) ENGINE=MyISAM;
Query OK, 0 rows affected (0.03 sec)
```

```
mysql> INSERT INTO t_myisam VALUES(1);
Query OK, 1 row affected (0.02 sec)
```

观察查询该表的执行计划:

```
mysql> EXPLAIN SELECT * FROM t_myisam;
+----+-------------+----------+------------+--------+---------------+------+---------+------+------+----------+-------+
| id | select_type | table    | partitions | type   | possible_keys | key  | key_len | ref  | rows | filtered | Extra |
+----+-------------+----------+------------+--------+---------------+------+---------+------+------+----------+-------+
|  1 | SIMPLE      | t_myisam | NULL       | system | NULL          | NULL | NULL    | NULL |    1 |   100.00 | NULL  |
+----+-------------+----------+------------+--------+---------------+------+---------+------+------+----------+-------+
1 row in set, 1 warning (0.00 sec)
```

可以看到,`type`列的值为`system`

注: 若存储引擎为InnoDB,则即使表中只有1条记录,也不会显示`system`,因为使用InnoDB存储引擎的表,其统计数据是不精确的.

```
mysql> CREATE TABLE t_innodb (a INT);
Query OK, 0 rows affected (0.03 sec)
```

```
mysql> INSERT INTO t_innodb VALUES(1);
Query OK, 1 row affected (0.01 sec)
```

```
mysql> EXPLAIN SELECT * FROM t_innodb;
+----+-------------+----------+------------+------+---------------+------+---------+------+------+----------+-------+
| id | select_type | table    | partitions | type | possible_keys | key  | key_len | ref  | rows | filtered | Extra |
+----+-------------+----------+------------+------+---------------+------+---------+------+------+----------+-------+
|  1 | SIMPLE      | t_innodb | NULL       | ALL  | NULL          | NULL | NULL    | NULL |    1 |   100.00 | NULL  |
+----+-------------+----------+------------+------+---------------+------+---------+------+------+----------+-------+
1 row in set, 1 warning (0.00 sec)
```
